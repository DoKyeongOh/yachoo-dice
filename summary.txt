일단 설계를 할 때, ER모델 -> DFD 순으로 그려야할듯.
ER모델을 그리는 과정에서 어떤 엔티티가 있는지 알기위해 플로우 차트를 만들어보는것도 괜찮을듯


0. 엔티티를 나누기전에 야추다이스라는 게임의 사전 조사를 해보자.
    - 야추다이스 : 주사위 다섯 개를 굴려서 나온 눈의 조합으로 점수를 내고, 턴제로 진행하며 이 점수를 합산하여 최종적으로 주사위 눈의 수가 가장 많은 사람이 이긴다.
    - 여러 명이 할 수 있는 게임이다.
    - 시중에서는 두 명이 하는게 일반적이지만, 여러명이해도 무방해보인다.

0.1 야추다이스의 마이너 조합
    - 1 조합 : 숫자 1의 주사위 눈 총합, 0~5 가능
    - 2 조합 : 숫자 2의 주사위 눈 총합, 0~10 가능
    - 3 조합 : 숫자 3의 주사위 눈 총합, 0~15 가능
    - 4 조합 : 숫자 4의 주사위 눈 총합, 0~20 가능
    - 5 조합 : 숫자 5의 주사위 눈 총합, 0~25 가능
    - 6 조합 : 숫자 6의 주사위 눈 총합, 0~30 가능

0.2 야추다이스의 메이저 조합 (만들기 나름일수도 있음)
    - 3 of a kind (스리 오브 어 카인드) : 같은 주사위 눈의 종류가 3개 이상인 경우의 조합
    - 4 of a kind (포 오브 어 카인드) : 같은 주사위 눈의 종류가 4개 이상인 경우의 조합
    - 풀하우스 : 같은 주사위 눈의 종류가 3개, 또 다른(혹은 같은) 종류의 주사위가 2개(남은 것 전부)인 경우의 조합
    - 스몰 스트레이트 : 주사위 눈의 종류가 연속적으로 4종류 이상인 경우의 조합
    - 라지 스트레이트 : 주사위 눈의 종류가 연속적으로 5종류 이상인 경우의 조합
    - 야추 : 주사위 눈의 종류가 모두 같은 경우의 조합
    - 찬스 : 모든 주사위의 총합

0.3 야추다이스의 추가 득점 방법
    - 마이너 조합의 합산 값이 나올 수 있는 점수의 일정 수준 이상을 넘긴 경우에 일정량의 점수를 추가적으로 획득할 수 있음

0.4 야추다이스 게임의 흐름
    - 모든 "플레이어" 의 플레이 순서를 결정
    - 각 플레이어는 라운드당 자신의 "턴(순서)"에 주사위를 굴릴 기회를 3회 얻음
    - 플레이어는 주사위를 굴릴 때마다 원하는 조합을 만들기 위해 주사위를 "킵" 할 수 있음
    - 플레이어가 주사위를 굴린 직후 원하는 조합을 획득하는 등의 이슈로 인해 턴을 종료할 수 있음
    - 플레이어의 턴이 종료될 때에는 무조건 본인이 야추다이스의 여러 "주사위 조합" 중 하나를 선택해야함
    - 최종 라운드가 종료된 직후에는 플레이어가 획득한 점수를 각각 합산하고 합산된 점수를 사용하여 등수 계산 및 우승자 도출

1. 엔티티는 뭐가 있을까?
    - 유저 (야추다이스를 플레이하기 위한 유저)
        - 유저 고유 ID 또는 "ID & PW"
        - 회원가입일
    - 플레이어 프로필
        - 프로필 ID
        - 프로필 사진 (추후 추가)
        - 플레이어 닉네임
        - 유저 ID
        - 플레이어 전적
            - 게임 우승 횟수
            - 게임 플레이 횟수
    - 게임
        - 게임 ID
        - 게임 이름 (방 이름)
        - 게임 스펙
            - 참가 유저 ID 리스트 (참가 명단) => 명시적인 유저 턴 순서가 필요함
            - 턴 순서
            - 총 라운드 == 모든 주사위 조합의 수
        - 현재 라운드
        - 현재 턴
    - 주사위 조합
        - 속성 (type) : 메이저, 마이너
    - 게임 결과
        - 순위 리스트
1.1 대략적인 시나리오
    (1) 게임 생성 : ID, 이름 필수, 라운드는 0
    (2) 플레이어 참가 : 게임에 참가 플레이어 명단에 플레이어 ID 추가
    (3)

1.2 주사위를 굴릴 때마다 해당 주사위 결과를 통해 각 조합에서 얻을 수 있는 점수를 알아야함
